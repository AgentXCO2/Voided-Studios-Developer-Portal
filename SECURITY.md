# 🔐 Security Policy — Voided Studios Dev Page

## Supported Versions

We actively maintain and provide security updates for the following versions of the Dev Page system.

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | ✅ Active Support   |
| 1.9.x   | ⚠️ Limited Support  |
| 1.0.x   | ❌ No Support       |
| < 1.0   | ❌ Unsupported      |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability in the Voided Studios Dev Page system, please report it responsibly.

### 📩 Where to report:
- GitHub Issues (preferred for non-sensitive bugs)
- Direct message to the project maintainer (for sensitive issues)

---

### ⏱ Response Time

- Initial response: within **24–72 hours**
- Status update: within **3–5 days**
- Resolution time: depends on severity

---

### 🧠 What happens after you report:

1. We review and confirm the vulnerability
2. We assess severity and impact
3. We patch the issue (if valid)
4. We release a fix update if needed
5. Credit may be given (if requested)

---

### ❌ What to avoid:

- Do NOT publicly exploit vulnerabilities
- Do NOT access or modify other users’ data
- Do NOT spam or mass-report non-issues

---

## 🔒 Security Philosophy

Voided Studios prioritizes:
- Data integrity
- Admin control safety
- Minimal attack surface
- Fast patch response

---

## 🧩 Scope

This policy applies to:
- Dev Admin Panel
- Firestore integration
- Public Dev Display Site
- Authentication system (if enabled)

---

Thank you for helping keep Voided Studios secure.
